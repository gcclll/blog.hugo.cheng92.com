Vue3 Reactivity

从零开始实现 vue3 的 reactivity 模块，基于 Proxy + Reflect api 实现响应式。大致 实现路径 reactive -> proxy 开启代理 -> reflect 映射到原子操作，监听 get/set 执行 track & trigger 收集信赖。

注意：这里会省略部分不影响理解的代码，和一些非法参数等等的检查，目的是为了减少代码量，但不影响学习和理解源代码


1. READY(handlers)

由于 Proxy + Reflect 并没有针对集合类型(Set, Map)的数据有直接的映射接口，因此对 于这两类数据要做特殊处理，直接通过 getter, setter 方法来变相实现 reactive 功能。

所以这里用到两种 handlers， 一个是 mutableHandlers 针对普通类型的对象，一个是 mutableCollectionHandlers 集合类型的对象。

而 handlers 是传递给 Proxy 接口的一个参数，具体的使用要参考 Proxy + Reflect 的使用。

当传递一个对象 target 给 Proxy 的时候，它会返回一个代理后的对象，对这个对象 做的一些原子操作会被 handlers 里面提供的方法所拦截，然后在这些 handlers 对应的方 法里面又可以通过 Reflect 来执行原子操作(比如：赋值和取值)，这样就能监听某一 个对象的原子操作在它们变更时来执行一些我们所想要的操作。

更多与 proxy 和 reflect 相关的内容请点击 JavaScript Proxy And Reflect。

参数说明：

name description
target object to be reactive
isReadonly target is readonly ?
baseHandlers a param to Proxy for normal objects(Object, Array)
collectionHandlers a param to Proxy for collectional objects(Map, Set)
proxyMap cache target=>proxy relationship, avoid proxy repeatly 
function createReactiveObject(
  target,
  isReadOnly,
  baseHandlers,
  collectionHandlers,
  proxyMap
) {

  // 不用对已经代理过的对象重复进行代理
  const existingProxy = proxyMap.get(target)
  if (existingProxy) {
    return existingProxy
  }

  // targetType 是 target 原始类型 Object.prototype.toString.call(target) => [object Object]
  // 中的 "Object", 这里如果是 Map/Set/WeakMap/WeakSet 则使用 collectionHandlers 如果是 Object/Array
  // 就使用 baseHandlers(TIP:这里路过非法检测)
  const proxy = new Proxy(target, isCollection(target) ? collectionHandlers : baseHandlers)
  proxyMap.set(target, proxy)
  return proxy
}

5.1.1. createGetter with track

function createGetter(isReadonly = false, shallow = false) {

  return function get(target, key, receiver) {
    if (key === '__v_isReactive') {
      // 检测是不是一个 reactive 对象时
      return !isReadonly
    } else if (key === '__v_isReadonly') {
      // 检测是不是一个只读对象时，如： 取target.__v_isReadonly 的值
      return isReadonly
    } else if (key === '__v_isShallow') {
      return shallow
    } else if (key === '__v_raw' && receiver === (
      isReadonly
        ? shallow ? shallowReadonlyMap : readonlyMap
      : shallow ? shallowReactiveMap : reactiveMap
    ).get(target)) {
      // 读原始对象，代理之前的
      return target
    }

    // 通过原子操作得到真实的值
    const res = Reflect.get(target, key, receiver)

    // 只读的是不能设置值的因此也没必要去做多余的监听
    if (!isReadonly) {
      track(target, 'get', key)
    }

    console.log(`createGetter| get, key=${key}, value=${res}`)

    if (isObject(res)) {
      // TODO readonly(res)
      return isReadonly ? res : reactive(res)
    }

    return res
  }
}

    console.log(`createInstrumentationGetter| key=${key}`) // 这里应该输出集合方法名，如：has,set,get

    // 取被代理之前的原始对象
    if (key === '__v_raw') {
      return target

      key,
      receiver

reactive 中订阅发布逻辑相关的三大函数：

track: get 取值操作时去收集当前的函数(也就是这个变量的监听者其实就是一个函数，只 不过被封装成了一个 ReactiveEffect 类。)

trigger: set 设值操作时会去触发 track 收集到的信赖并执行它们，这个执行的动作实际 是由 Scheduler 这个模块去管理的，这里就不深入探讨这个了有兴趣 的同学可以参考 Vue3 功能拆解② Scheduler 渲染机制 - 若叶知秋 2 或 TODO:build you own vue scheduler。

effect: 将一个函数封装成 ReactiveEffect 类，调用的时候会立即执行一个 fn 收集信赖。

要搞清楚是怎么回事，还需要从头开始，也就是 reactive -> createReactiveObject 会发 生这两个函数里面并没有与这两相关的代码，那么只能到 mutableHandlers 和 mutableCollectionHandlers 中去找了，因为这两个涉及的是取值和 设值两个行为，所以下面直接将涵盖的相关函数列出来。

  1. baseHandlers.ts 对应 mutableHandlers, 中包含 get, set, deleteProperty, has, ownKeys

    其中 get, has, ownKeys 属于取值行为，所以应该调用 track 。

    而 deleteProperty, set 为设置值行为，所以应该调用 trigger 。

  2. collectionHandlers.ts 对应 mutableCollectionHandlers

    因为 collection 都是通过对集合方法名的 get 来实现代理的，这里就直接进行代理后 的行为函数判定，也就是包含 get, size, has, add, set, delete, clear, forEach 方法和属性(size)。

    其中 get, get size, has, forEach 均为取值行为，所以应该调用 track 。

    而 set, delete, clear 为设值改变行为，所以应该调用 trigger 。

    这里就涉及到的一个经典问题：就是能不能在 effect->fn 中再次调用 effect(fn) ，在 不加控制情况下就会死循环，虽然每次执行 effect.run->fn 在 try 中都会清理一次 effects 信赖，但是在 fn 中调用 effect(fn) 的话会导致重新收集了 fn 进 reactive 变量的信赖表。所以在 trigger->triggerEffects 中增加了一个选项来控制是不是要放 开这个限制(即可以在 fn 中改变 reactive 变量同时调用 effect(fn) ，前提是你知道自己在干什么，并做好防控避免死循环)，这个选项就是 triggerEffects-allowRecurse 它默认是关闭的如果要开启需要 effect(fn, { allowRecurse: true })
